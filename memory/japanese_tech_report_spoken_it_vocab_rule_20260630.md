# Japanese tech report spoken IT vocabulary rule

User preference was clarified for `skills/japanese-tech-report/`:

- Spoken customer/onsite Japanese should still use common IT vocabulary actively, such as `取得`, `実施`, `確認`, `反映`, `更新`, `影響範囲`, `切り分け`, `デグレ`, `リリース判定`, `ログ監視`, `ステージング環境`, and `本番環境`.
- The issue to avoid is not kanji or IT terminology. The issue is stiff written grammar in meeting scripts.
- Avoid spoken output with chains like `〜を行い、〜します`, `〜しており、〜です`, `〜した上で、〜します`, `〜であり、〜です`, `〜に伴い、〜が必要になります`, and long `〜ため、〜し、〜します`.
- Prefer spoken rhythm: `まず〜します。`, `そのあと〜を確認します。`, `確認できたら〜を共有します。`, `理由は〜です。なので〜します。`
- Avoid overusing `〜と見ています` in spoken meeting scripts. Prefer `今のところ〜です`, `〜かなと思っています`, `〜そうです`, `〜なさそうです`, or `〜でよさそうです`.

The skill was updated to include an IT vocabulary policy, avoid-word list, spoken grammar policy, and judgment wording alternatives.
